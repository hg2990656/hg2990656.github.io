Babel 7

发表于2021-01-06|更新于2021-01-07|前端
|字数总计:3.3k|阅读时长:16分钟|阅读量:

Babel 是一个工具集，主要用于将 ES6 版本或更高版本的 javaScript 代码转为 ES5 等向后兼容的 javaScript 代码，从而可以运行在低版本浏览器或其它环境中。

详细的 Babel 7 教程：Babel 教程，在此基础上整理 Babel 中涉及的包之间的关系以及 vue-cli 脚手架中 Babel 的相关依赖。

Babel 组成

Babel 的主要工作有两部分：

  • 语法转换：使用基础 npm 包进行语法转换，例如：把 ES6 的箭头函数语法转换为 ES5 的函数定义语法。 箭头函数语法、class 定义类语法和解构赋值等等都是 ES6 新增的语法。
  • 补齐 API：通过 Polyfill 的方式在目标环境中添加缺失的特性，例如：ES6 中的 Promise。API 指别人已经写好的对象、类、函数或方法，新的 API 分两类：
    • Promise、Map、Symbol、Proxy、Iterator 等全局对象及其对象自身的方法，例如：Promise.resolve
    • 新的实例方法，例如：数组实例方法 Array.from()、Array.of()、flat()，Object.assign()

Babel 涉及的依赖关系如图，省略了@babel/cli：

  • @babel/core：babel 的核心包。

  • @babel/preset-env：插件的集合，通过设置参数项可以进行针对性语法转换以及 polyfill 的部分引入，可直接搭配core-js@3和regenerator/runtime实现全部的功能，但是会在每个文件中都引入语法转换的辅助函数，因此通常会搭配@babel/runtime和@babel/plugin-transform-runtime使用避免重复引入辅助函数。

  • @babel/polyfill（已废弃）：仅仅是引用core-js、regenerator-runtime这两个npm包，core-js是实现 ployfill 的核心包，regenerator-runtime包含async/await转换为generator的 ployfill实现。

    @babel/ployfill使用的core-js为2.x.x版本，而core-js本身已发布了3.x.x版本，@babel/polyfill已被废弃，以后也不会使用3.x.x版本的包了，新版本的core-js@3实现了许多新的功能，例如：数组的 includes 方法，Babel 7 建议单独使用两个包。

  • @babel/runtime：包含两个文件夹：helpers（定义了处理新的语法关键字的辅助函数）、regenerator（仅仅是引用regenerator-runtime）。

  • @babel/runtime-corejs3：@bable-runtime 的加强版，选择其中一种安装即可，包内有四个文件夹：core-js（引用core-pure-js/feature下的ployfill实现）、core-js-stable（引用core-pure-js/stable下的ployfill实现）、helpers（定义了处理新的语法关键字的辅助函数）、regenerator（仅仅是引用 regenerator-runtime）。

  • @babel/plugin-transform-runtime：包含三个功能：

    • 配合@babel/runtime或者@babel/runtime-corejs3中任意一个包，自动移除语法转换后内联的辅助函数，使用@babel/runtime/helpers或@babel/runtime-corejs3/helpers中的辅助函数来替代，配置如下：

    • 安装 @babel/runtime-corejs3 且设置插件配置的 core-js 为 3，当代码里使用了core-js的 API，自动引入 @babel/runtime-corejs3/core-js-stable/来替代@babel/preset-env转换过程中使用core-js/stable生成的全局变量，前者使用core-js-pure，后者使用core-js，core-js-pure是core-js的无污染版本。

    • 当代码里使用了 async 函数，自动引入@babel/runtime/regenerator或者@babel/runtime-corejs3/regenerator来替代@babel/preset-env转换过程中使用regenerator-runtime/runtime生成的全局变量。

因此实际项目中 Babel 配置方案如下：

@babel/preset-env

方案一：使用 @babel/preset-env + corejs@3 + regenerator-runtime（不使用 @babel/runtime）

方案二：使用 @babel/preset-env + @babel/runtime（包含regenerator-runtime依赖） + @babel/plugin-transform-runtime + corejs@3

方案三：使用 @babel/preset-env + @babel/runtime（包含 regenerator-runtime 依赖） + @babel/plugin-transform-runtime + @babel/runtime-corejs3（包含 core-js-pure 和 regenerator-runtime 依赖）

方案一和方案二实际使用的是 core-js，采用修改全局对象的方式 polyfill，vue-cli 采用方案二。

方案三使用的是 core-js-pure，采用不污染全局对象模式 polyfill，一般用于工具库的开发。

@vue/cli-plugin-babel

@vue/cli-plugin-babel默认使用 Babel 7 + babel-loader + @vue/babel-preset-app，但是可以通过 babel.config.js 配置使用任何其它 Babel 预设选项或插件。

cache-loader 默认开启，被缓存的东西存储在 /node_modules/.cache/babel-loader。

thread-loader 会在多核机器上默认开启。你可以在 vue.config.js 中设置 parallel: false 将其关闭。

@babel/runtime-corejs3

安装@babel/runtime-corejs3

@babel/runtime-corejs3依赖的是core-js-pure而不是core-js，因此安装@babel/runtime-corejs3时会自动安装core-js-pure和regenerator-runtime，而安装@babel/runtime时也会安装依赖regenerator-runtime，因此若已经安装了@babel/runtime，那么安装@babel/runtime-corejs3时只会安装core-js-pure包。

core-js-pure

It’s a version without global namespace pollution.

它是一个没有全局命名空间污染的 core-js 版本。
